## 核心流程

### 1. 录音与识别流程

```
用户按住 Fn 键
       │
       ▼
CGEventTap 捕获 keyDown 事件
       │
       ▼
GlobalEventMonitor 通知 AppDelegate
       │
       ▼
AppDelegate 显示 FloatingWindow
       │
       ▼
SpeechRecognizer 开始录音
       │
       ▼
实时音频 → RMS 计算 → WaveformView 更新
       │
       ▼
用户松开 Fn 键
       │
       ▼
SpeechRecognizer 停止录音
       │
       ▼
获取识别结果 → LLMRefiner 润色（可选）
       │
       ▼
TextInjector 注入文本到当前输入框
```

### 2. 文本注入流程

```
获取识别文本
       │
       ▼
检测当前输入法是否为 CJK
       │
       ├── 是 → 临时切换到 ASCII 模式
       │           │
       │           ▼
       │      复制文本到剪贴板
       │           │
       │           ▼
       │      模拟 Cmd+V 粘贴
       │           │
       │           ▼
       │      恢复 CJK 输入法
       │
       └── 否 → 直接复制粘贴
```

## 模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| AppDelegate | 应用生命周期管理、协调各模块 | 所有模块 |
| GlobalEventMonitor | 全局 Fn 键监听和事件处理 | CoreGraphics |
| SpeechRecognizer | 语音识别、音频录制、RMS 计算 | Speech, AVFoundation |
| FloatingWindow | 浮动窗口显示、动画管理 | AppKit |
| WaveformView | 波形可视化、动画渲染 | AppKit |
| TextInjector | 剪贴板操作、输入法切换、按键模拟 | Carbon, AppKit |
| LLMRefiner | AI 润色 API 调用、结果处理 | Foundation |
| SettingsWindow | 设置界面、配置持久化 | AppKit |
| LanguageManager | 语言切换、本地化支持 | Foundation |
| Logger | 日志记录、文件管理 | Foundation |

## 安全与隐私

### 权限需求

| 权限 | 用途 | 请求时机 |
|------|------|----------|
| 麦克风 | 语音识别 | 首次录音 |
| 辅助功能 | 全局快捷键、文本注入 | 首次启动 |

### 数据安全

- 语音数据仅在本地处理（Apple Speech API）
- LLM API 调用仅在用户启用时发生
- 配置数据存储在 UserDefaults，不加密
- 日志文件保存在用户目录，包含调试信息

## 扩展性设计

### 添加新语言

1. 在 `LanguageManager` 中添加语言代码
2. 添加本地化字符串
3. 更新语言菜单

### 支持新的 LLM 提供商

当前仅支持 OpenAI 兼容 API。如需添加 Anthropic Claude：

1. 创建 `LLMProvider` 协议
2. 实现 `OpenAIProvider` 和 `ClaudeProvider`
3. 在设置中添加提供商选择

### 自定义快捷键

当前硬编码为 Fn 键。扩展方案：

1. 在设置中添加快捷键配置
2. 使用 `MASShortcut` 或类似库
3. 动态更新 CGEventTap 监听

## 性能考虑

### 内存管理

- `AppDelegate` 使用全局变量避免提前释放
- CGEventTap 在应用生命周期内保持运行
- 音频缓冲区及时释放

### CPU 优化

- RMS 计算使用高效算法
- 动画使用 Core Animation 硬件加速
- API 调用使用后台线程

### 电池优化

- 仅在录音时活跃
- 后台时暂停事件监听
- 低功耗模式待实现
